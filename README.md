## Quick Start

### Basic Usage
```zig
const std = @import("std");
const abi = @import("abi");

pub fn main() !void {
    const allocator = std.heap.page_allocator;

    // Create AI agent using the exported AgentConfig
    var agent = try abi.features.ai.agent.Agent.init(allocator, .{ .name = "QuickStart" });
    defer agent.deinit();

    // Process a message
    const response = try agent.process("Hello!", allocator);
    defer allocator.free(@constCast(response));
}
```

> `Agent.process` duplicates its reply into the allocator you provide, so callers must free the returned buffer once they're
> done with it.

### Vector Database
```zig
// Open database
var db = try abi.features.database.database.Db.open("vectors.wdbx", true);
defer db.close();

// Add and search vectors
const embedding = [_]f32{0.1, 0.2, 0.3};
const results = try db.search(&embedding, 10, allocator);
```

### WDBX CLI (roadmap)

Dedicated WDBX commands are part of the planned CLI experience and are being prototyped in `src/tools/main.zig`. The installed executable built from `src/main.zig` does not currently expose these subcommands.

## Modules

- **`core/`**: Core utilities and framework foundation
- **`framework/`**: Runtime orchestrator that wires features and plugins together
- **`ai/`**: AI agents and machine learning components
- **`database/`**: WDBX-AI vector database with HNSW indexing
- **`net/`**: HTTP/TCP servers and client libraries
- **`simd/`**: SIMD-accelerated operations
- **`perf/`**: Performance monitoring and profiling
- **`gpu/`**: GPU acceleration and rendering
- **`ml/`**: Machine learning algorithms

## CLI

### Default executable (`src/main.zig`)

The `zig build` output initialises the framework, captures its human-readable startup summary, and writes both the banner and summary to standard output before exiting. No runtime flags or subcommands are currently recognised.

```bash
zig build
./zig-out/bin/abi
```

This prints `ABI Framework bootstrap complete` followed by the structured framework summary generated by `framework.writeSummary`.

### Experimental CLI prototype (`src/tools/main.zig`)

An expanded command-line interfaceâ€”including GPU, database, and chat helpersâ€”is under active development. To try it locally, invoke the tool directly via Zig:

```bash
zig run src/tools/main.zig -- --help
```

This command builds the prototype on the fly and shows the available experimental subcommands. The tool is not packaged or installed by default; expect frequent breaking changes while it incubates.
