Simple, Powerful API

Get started with just a few lines of code. The unified configuration system and builder pattern make it easy to enable exactly the features you need.

  • Unified configuration with builder pattern
  • Feature-gated compilation for minimal binaries
  • Runtime feature toggling via CLI flags
  • Graceful degradation with fallback systems
  • Type-safe API with compile-time checks
main.zig 
const std = @import("std");
const abi = @import("abi");

pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    // Configure with builder pattern
    const config = abi.Config.init()
        .withAI(true)
        .withGPU(true)
        .withDatabase(true);

    var framework = try abi.Framework.init(allocator, config);
    defer framework.deinit();

    // Access AI features
    if (framework.ai()) |ai| {
        const response = try ai.generate("Hello!");
        std.debug.print("{s}\n", .{response});
    }
}
Quick Start

Up and Running in Minutes

Follow these simple steps to start building with ABI Framework.

1

Clone the Repository

Get the latest version from GitHub.

git clone https://github.com/donaldfilimon/abi.git
2

Build the Project

Compile with your desired features enabled.

zig build -Denable-ai=true -Denable-gpu=true
3

Run the Tests

Verify everything is working correctly.

zig build test --summary all
4

Launch the TUI

Explore the interactive interface.

zig build run -- tui
CLI

Powerful Command Line

Full-featured CLI for database operations, AI agents, GPU management, and more.

$ zig build run -- tui Interactive UI
$ zig build run -- agent AI Agent Mode
$ zig build run -- db stats Database Stats
$ zig build run -- gpu backends GPU Backends
$ zig build run -- llm chat model.gguf Chat with LLM
$ zig build run -- train run --epochs 10 Training
$ zig build run -- --list-features List Features
$ zig build run -- system-info System Info
