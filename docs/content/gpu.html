The GPU module provides a unified backend interface for acceleration. Backends can be selected at build time and discovered at runtime.

Backend selection

zig build -Dgpu-backend=vulkan
zig build -Dgpu-backend=cuda,vulkan
zig build -Dgpu-backend=auto
zig build -Dgpu-backend=none

Available backends

  • none - Disable all GPU backends
  • auto - Auto-detect available backends
  • cuda - NVIDIA CUDA (requires NVIDIA drivers + toolkit)
  • vulkan - Vulkan (cross-platform)
  • metal - Metal (macOS only, includes Accelerate framework)
  • stdgpu - Standard GPU fallback
  • webgpu - WebGPU for browser/WASM targets
  • opengl - OpenGL
  • opengles - OpenGL ES (mobile)
  • webgl2 - WebGL 2.0
  • fpga - FPGA backend

Multiple backends can be enabled with comma-separated values: -Dgpu-backend=cuda,vulkan

Runtime behavior

  • Automatic fallback to CPU when a backend is unavailable
  • Explicit backend selection via ABI_GPU_BACKEND environment variable
  • Memory pools tuned for large model workloads

GPU health checks

zig build run -- gpu status
zig build run -- gpu devices
zig build run -- gpu backends
zig build run -- gpu summary
