ABI uses a unified configuration struct with optional feature blocks. The builder pattern keeps configuration readable and composable across modules.

Builder pattern

const abi = @import("abi");
const config = abi.Config.init()
    .withAI(true)
    .withGPU(true)
    .withDatabase(true);

var framework = try abi.Framework.init(allocator, config);
defer framework.deinit();

Config loader

Use the ConfigLoader to merge environment variables and defaults with a base config. This is useful in CLI tools and services. 

const config_mod = @import("abi").config;
var loader = config_mod.ConfigLoader.init(allocator);
defer loader.deinit();
const config = try loader.load();

Environment overrides

Common overrides include GPU backend selection, model paths, and database locations. Export these before starting your service. 

export ABI_GPU_BACKEND=vulkan
export ABI_LLM_MODEL_PATH=/models/llama.gguf
export ABI_DB_PATH=/data/abi.db
