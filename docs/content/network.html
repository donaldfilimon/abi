The network module enables distributed compute and coordination. It includes service discovery, node registration, and Raft based consensus primitives.

Use cases

  • Sharded workloads across multiple nodes
  • Shared registries for capabilities and metadata
  • Fault tolerant coordination for long running jobs

CLI entry points

zig build run -- network list
zig build run -- network status

Operational notes

Distributed features are disabled by default in minimal builds. Enable the network feature flag when you need clustering support.
