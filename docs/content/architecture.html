ABI is structured around feature modules that expose stable APIs. Modules can be enabled or disabled at build time without changing the public surface.

Module layout

  • Core runtime: scheduling, concurrency, memory
  • AI module: agents, LLMs, streaming, training
  • GPU module: backend selection and kernel execution
  • Database module: vector storage and indexing
  • Network module: distributed compute and coordination

Stub pattern

Each feature module has a real implementation and a stub. When a feature is disabled, the stub keeps APIs intact and returns a feature disabled error. This keeps builds predictable and avoids conditional imports in user code.

Configuration and registry

The Config struct drives framework initialization. A registry layer handles feature lifecycle and validation, ensuring modules are started in the correct order and shut down cleanly.
