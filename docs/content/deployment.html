ABI can run as a local CLI, a long running service, or inside containers. The deployment path depends on which modules you enable.

Docker

docker compose up -d abi
docker compose up -d abi-gpu

Feature tuning

Trim unused modules at build time and keep runtime configuration minimal for production. Use environment variables to keep secrets out of config files. 

zig build -Denable-ai=true -Denable-network=false
export ABI_MASTER_KEY="change-me"

Operational checklist

  • Pin GPU backends to known compatible drivers
  • Store model and database files on fast disks
  • Enable observability before exposing public endpoints
