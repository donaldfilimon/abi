## Recommended sequence (when approved)

1. Keep `modernized/` as documentation pointers only.
2. Extract one leaf package at a time **inside `src/`** (or a tracked `packages/` under the existing build) with parity tests — prefer Approach-1 style extraction over a clean-slate tree.
3. Only after several packages prove green, decide whether `modernized/` becomes the build root or remains historical.
4. Delete or archive unused pointer READMEs after cutover; never force-push `main`.
