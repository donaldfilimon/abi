# Hardware Acceleration Exploration (FPGA / ASIC)

## Executive summary
This document surveys options for hardware acceleration beyond GPUs for ABI workloads, focusing on FPGA and ASIC approaches, and provides a recommended next-steps plan and PoC checklist.

## Motivation
Specialized hardware can potentially reduce latency, increase throughput, and improve power efficiency for inference workloads. This doc evaluates feasibility, tooling, and a sensible PoC roadmap.

## FPGA
- Toolchains: Xilinx (Vivado, Vitis), Intel (Quartus), open-source (SymbiFlow)
- Workflows: HLS for algorithmic design, or RTL for performance-critical kernels
- Integration: host <-> FPGA communications (PCIe, NVLink), driver support

## ASIC
- Survey: TPU, Groq, Cerebras, Habana (Intel)
- Considerations: NRE costs, design time, vendor partnerships, software stack

## PoC plan
1. Short-list candidate targets (e.g., Xilinx Alveo, Intel Stratix, or cloud providers offering FPGA instances)
2. Implement a small kernel (e.g., matrix multiply or attention inner product) as an FPGA HLS demo
3. Run functional verification and collect latency/throughput metrics
4. Compare to GPU baseline for same workload

## Benchmarking plan
- Metrics: latency p50/p90/p99, throughput (tokens/sec), memory usage, power consumption
- Dataset: representative inference requests (short/medium/long prompts)
- Reporting: JSON output compatible with `tools/benchmark-dashboard` sample schema

## Risk & Cost
- FPGA: faster time-to-prototype but lower peak perf vs purpose ASICs; moderate cost for dev boards
- ASIC: high NRE and lead time; high potential perf but only sensible with established product roadmap

## Recommendations
- Short term: prototype an HLS kernel on a cloud FPGA instance or affordable dev board
- Medium term: identify a partner or vendor for deeper exploration if PoC shows clear perf/cost benefits
