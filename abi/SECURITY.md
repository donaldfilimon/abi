# Security Policy

## Reporting a Vulnerability
If you discover a potential security vulnerability, please report it privately to the maintainers.

- Preferred contact: open a private issue in this repository and add `security` label, or email the maintainers (contact details TBD).
- Provide a clear description of the issue, steps to reproduce, affected versions, and any PoC if possible.

## Response Process
- We will acknowledge receipt within 72 hours.
- We aim to provide an initial triage and expected remediation timeline within 7 days.

## Disclosure Policy
- We follow coordinated disclosure practices. Do not publish exploit details until the issue has been resolved or an agreed disclosure timeline elapsed.

## Recommended tools
- We recommend running dependency scanning (CodeQL, Dependabot, or Snyk) and keeping third-party libs up to date.
