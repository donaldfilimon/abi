# ABI Architecture & Module Guidelines

## Framework Overview

ABI is a modular Zig 0.16 framework for distributed AI systems, vector search, and GPU compute. It uses comptime feature gates controlled via `build.zig` flags.

## Named Modules (dev.2905+)

Only 3 named modules exist:

| Module | Root Source | Purpose |
|--------|-----------|---------|
| `abi` | `src/root.zig` | **All** source files under `src/` belong to this single module |
| `build_options` | (synthetic) | Comptime feature flags generated by `build/modules.zig` |
| `cli` | `tools/cli/mod.zig` | CLI tool module (imports `abi`) |

The `foundation` namespace (`abi.foundation`) is part of the `abi` module, accessed via `src/services/shared/mod.zig`. It is **not** a separate named module.

**Critical rule**: Every `.zig` file belongs to exactly one module. All `src/` files are in the `abi` module. There are NO `shared_services`, `core`, or `wdbx` named modules.

## Directory Structure

```
src/
├── root.zig                    # Public package root (thin re-export)
├── abi.zig                     # Internal composition (comptime gating hub)
├── core/                       # Foundation: config, framework, registry, errors
│   ├── config/                 # Feature configs (always-on)
│   ├── framework/              # Lifecycle, feature_imports, builder
│   │   └── feature_imports.zig # Central comptime gating for 21 features
│   ├── registry/               # Plugin registry
│   └── database/               # Semantic store core (WDBX V3)
├── features/                   # Comptime-gated domain implementations
│   ├── ai/                     # LLM, agents, training, reasoning (mod.zig + stub.zig)
│   ├── gpu/                    # GPU compute, DSL, backends (mod.zig + stub.zig)
│   └── ... (19 feature dirs)
├── services/                   # Runtime services
│   ├── shared/                 # SIMD, time, sync, security, logging (foundation module root)
│   ├── runtime/                # Task scheduling, concurrency
│   ├── connectors/             # LLM providers (23 integrations)
│   └── ha/, lsp/, mcp/, acp/, platform/, tasks/
└── api_server/, inference/
tests/                             # Future integration & e2e tests
bindings/                          # Language bindings (moved from src/bindings/)
```

## Import Rules

- **External consumers** (CLI, tests, examples): `@import("abi")`
- **Within `src/`**: Use relative paths with explicit `.zig` extensions:
  ```zig
  const config = @import("../config/mod.zig");         // relative within src/
  const simd = @import("../../services/shared/simd/mod.zig");  // cross-zone OK (same module)
  ```
- **Foundation namespace**: Access via `@import("abi").foundation` (external) or relative `@import("../../services/shared/mod.zig")` (internal) — no separate named module
- **Feature flags**: `@import("build_options")` (named module, available everywhere)
- **NEVER** use `@import("shared_services")` or `@import("core")` — these named modules no longer exist
- Enforced by `zig build check-imports`

## Feature Flag Conventions

- Prefix: `feat_<name>` (NOT `enable_<name>`), all default to `true`
- 27 flags in `build/options.zig`, 56 combos validated in `build/flags.zig`
- Flags exported via `build_options` module
- Disable: `zig build -Dfeat-gpu=false`

## mod/stub Contract

Every comptime-gated feature has paired files:
- `mod.zig` — real implementation (compiled when feature enabled)
- `stub.zig` — empty implementation (compiled when feature disabled)

**Stubs must match mod public signatures exactly.** The compiler enforces this via dual imports in `src/core/framework/feature_imports.zig`.

## Key Module Map

| Public Path | Entry File | Purpose |
|-------------|-----------|---------|
| `abi.App` / `abi.AppBuilder` | `src/core/framework.zig` | Primary runtime |
| `abi.config` | `src/core/config/mod.zig` | Unified configuration |
| `abi.gpu` | `src/features/gpu/mod.zig` | GPU compute |
| `abi.ai` | `src/features/ai/mod.zig` | LLM, agents, training |
| `abi.database` | `src/features/database/mod.zig` | Semantic store facade |
| `abi.foundation` | `src/services/shared/mod.zig` | SIMD, time, sync, security |
| `abi.runtime` | `src/services/runtime/mod.zig` | Task scheduling |

## Build System

- `build/modules.zig` — Module creation (`wireAbiImports(module, build_opts)` wires `build_options`; no separate foundation module)
- `build/module_catalog.zig` — Feature module registration and catalog metadata
- `build/options.zig` — Feature flag definitions (27 `feat_*` flags)
- `build/flags.zig` — Validation matrix (56 flag combos)
- `build/test_discovery.zig` — Feature test manifest (81+ entries)

## Performance & SIMD

- Use `services/shared/simd/mod.zig` for portable hardware acceleration
- Prefer `std.Io` patterns for non-blocking I/O
- Target `native` by default, validate for `wasm32-freestanding`
