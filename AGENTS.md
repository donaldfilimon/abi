# AGENTS.md — ABI Framework

Compact instructions for automation and rapid ramp-up.

## Toolchain & Build
- **Zig Pin**: `.zigversion` (Zig 0.17.0-dev.304+9787df942).
- **macOS**: Always use `./build.sh` (native linker; LLD fails on Darwin 25+).
- **Linux / older macOS**: Use `zig build` directly.
- **Core Commands**:
  - `./build.sh check` — Primary gate (test + lint + mod/stub parity).
  - `./build.sh cli` — Build CLI (`zig-out/bin/abi`).
  - `./build.sh mcp` — Build MCP server (`zig-out/bin/abi-mcp`).
  - `./build.sh full-check` — Full validation suite (identical to `check`; still does NOT run integration tests or benchmarks).
  - `zig build fix` — Auto-format sources.
  - `zig build lint` — Check formatting.
  - `zig build check-parity` — Verify `mod.zig` vs `stub.zig` API match (required after public API changes).
  - `zig build run` — Run the app.
  - `zig build test -- --test-filter "<pattern>"` — Run a single test.
  - `zig build test-integration` — Run integration tests.
  - `zig build benchmarks` — Run benchmark suite.
  - `./build.sh test --summary all` — Run all tests with summary.
  - **Note**: `check` step does NOT run integration tests or benchmarks; use `full-check` for the closest to CI (though `full-check` also excludes them).

## Architecture
- **Entry points**:
  - `src/main.zig` — CLI executable (delegates to `abi_cli/dispatch.zig`).
  - `src/mcp/main.zig` — MCP server (JSON-RPC 2.0, stdio or SSE). *Exception: Uses `@import("abi")` as it's the main entry point.*
  - `src/root.zig` — Library root, exported as `@import("abi")`.
- **Mod/Stub Pattern**: Each feature has `mod.zig` (real impl) and `stub.zig` (no-op when disabled). Changing public APIs requires updating **both**. Run parity check after.
- **Feature Flags** (all default `true` unless noted):
  - `feat-ai`, `feat-gpu`, `feat-accelerator`, `feat-shader`, `feat-mlir`, `feat-wdbx`, `feat-os-control`
  - `feat-tui` defaults `false`
  - `feat-mobile` defaults `false`
  - Override with `-Dfeat-<name>=false` or `-Dfeat-<name>=true`
- **GPU Backend Selection**: Runtime config in `src/config.zig` (`gpu_backend` field, default `.stdgpu`); not a build-time option.
- **Conditional imports** in `src/features/mod.zig` select mod vs stub based on `build_options`.
- **Plugin Registry**: Auto-generated by `tools/generate_plugin_registry.zig` before CLI build. **Do not edit `src/plugin_registry.zig` manually.**

## Import Rules
- **Within `src/`**: Use relative imports ONLY. Never `@import("abi")` — causes circular dependency.
- **Exception**: `src/mcp/main.zig` may use `@import("abi")` as it's an executable entry point.
- Always include `.zig` extension on path imports.

## Project Structure
- **src/** — Source code (use relative imports)
- **tasks/** — Project documentation: `lessons.md` (onboarding), `todo.md` (work tracking)
- **tools/** — Maintenance scripts (e.g., `generate_plugin_registry.zig`)

## Conventions
- **Naming**: `camelCase` functions/vars, `PascalCase` types, `SCREAMING_SNAKE_CASE` constants, `snake_case` enum variants.
- **Error Handling**: Silent empty catch blocks are forbidden in data access, inference, and persistence paths. Errors must be logged or propagated.
- **Strings in structs**: Always duplicate string literals (`. = "text".*`) if struct has `deinit()` to prevent use-after-free.
- **Zig 0.17 patterns**:
  - `pub fn main(init: std.process.Init) !void`
  - `ArrayListUnmanaged(T).empty` for initialization
  - `std.mem.trimEnd` (not `trimRight`)
  - `std.mem.splitScalar` / `splitAny` / `splitSequence` (not `split`)
  - Use `foundation.time.unixMs()` (not `std.time.milliTimestamp`)

## Testing
- Tests are inline `test {}` blocks — no separate `test/` directory.
- Each module **must** include `std.testing.refAllDecls(@This())` in its test block. (See src/features/ai/aviva/mod.zig for example)
- Default GPU backend is `stdgpu` (CPU fallback via `simulated`).
- **Known pre-existing failures** (not regressions):
  - 1 wdbx hash_map segfault in integration tests (not a regression; tracked separately)

## Operational
- **MCP Launcher**: Use `mcp/launcher.sh` for cross-platform execution. `.mcp.json` configures `./zig-out/bin/abi-mcp` with stdio.
- **Sync Primitives**: `foundation/sync.zig` provides `RwLock` and `SpinLock`.
- **CI**: Commits with task-marker comments are blocked.
- **Agent workflow**: 
  1. Read `tasks/lessons.md` (authoritative source for conventions) at session start
  2. Keep `tasks/todo.md` updated for non-trivial work
  3. Run `./build.sh check` to verify baseline before changes
  4. After any public API change, run `zig build check-parity`
  5. Before marking work complete, run appropriate validation gate (`check` or `full-check`)
  6. **Never use `rm`** — use safe alternatives only
  7. Use conventional commits

## Development Checklist
```
# Session start
1. Read tasks/lessons.md (authoritative source for conventions)
2. Check tasks/todo.md for current work
3. Run ./build.sh check (verify baseline)

# During development
4. For API changes: update BOTH mod.zig AND stub.zig, then run zig build check-parity
5. Test frequently: zig build test -- --test-filter "<pattern>"

# Before marking complete
6. Run ./build.sh check (or full-check for the same — parity is already in check)
7. Update tasks/todo.md
8. Use conventional commits
```
