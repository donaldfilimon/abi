Architecture

The framework is organized into five layers, from public API to shared utilities. Each layer builds on the previous one, providing a clean separation of concerns.

Public API

abi.zig - Entry point with curated re-exports.

abi.init(), abi.shutdown(), abi.version()

Framework

src/framework/ - Lifecycle management and orchestration.

Configuration, feature orchestration, and plugin system.

Compute Engine

src/compute/ - Parallel execution runtime.

Work-stealing scheduler, GPU integration, memory arenas.

Feature Stacks

src/features/ - Domain-specific modules.

AI, Database, GPU, Network, Monitoring, Web.

Shared Utilities

src/shared/ - Cross-cutting concerns.

Platform abstractions, SIMD, crypto, logging, filesystem.
